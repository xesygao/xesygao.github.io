zookeeper集群部署

配置说明

  • tickTime
    Zookeeper基本时间单位(ms)。心跳时间间隔等于tickTime；会话超时时间等于2倍tickTime。
  • dataDir
    用于存储内存数据库快照。在没设置dataLogDir的情况下，还存储Zookeeper操作的事务日志。
  • dataLogDir
    用于存储Zookeeper操作的事务日志。
  • clientPort
    客户端连接的端口
  • initLimit
    follower连接并同步到leader允许的最大时间间隔。数值为n个tickTime。
  • syncLimit
    follower服务器与leader服务器之间信息同步允许的最大时间间隔。数值为n个tickTime。

配置示例

tickTime=2000
initLimit=5
syncLimit=2
dataDir=/home/zookeeper/data
dataLogDir=/home/zookeeper/logs
clientPort=2181
server.1=172.16.40.201:2888:3888
server.2=172.16.40.202:2888:3888
server.3=172.16.40.203:2888:3888

端口说明
1.2181: 对client提供服务
2.2888: 集群内机器通讯使用
3.3888: 选举leader使用

安装部署

3台zk服务器为例：ServerA、ServerB、ServerC

  1. 分别在3台服务器创建dataDir，dataLogDir目录
    mkdir -p /home/zookeeper/data
mkdir -p /home/zookeeper/logs
  2. 分别在3台服务器下载并解压Zookeeper：下载 
    tar -xzf zookeeper-3.4.13.tar.gz
  3. 进入zookeeper目录，分别将 conf 目录下的 zoo_sample.cfg 复制一份重命名为 zoo201.cfg,zoo202.cfg,zoo203.cfg,并按照以上配置酌情修改。
    cp zoo_sample.cfg zoo201.cfg
cp zoo_sample.cfg zoo202.cfg
cp zoo_sample.cfg zoo203.cfg

  4. 分别在data目录下新建myid

    echo "1" > myid
echo "2" > myid
echo "3" > myid

    server.X 这个数字就是对应，data/myid中的数字。在3个server的myid文件中分别写入了1，2，3，那么每个server中的zoo.cfg都配 server.1, server.2,server.3就行了。

  5. 分别启动zookeeper

    bin/zkServer.sh start conf/zoo201.cfg
bin/zkServer.sh start conf/zoo202.cfg
bin/zkServer.sh start conf/zoo203.cfg
  6. 测试是否启动了该Server，若回复imok表示已经启动
    echo ruok|nc 172.16.40.201 2181
echo ruok|nc 172.16.40.202 2181
echo ruok|nc 172.16.40.203 2181

    常用命令

    连接到zk
    bin/zkCli.sh -server 172.16.40.201:2181
bin/zkCli.sh -server 172.16.40.202:2181
bin/zkCli.sh -server 172.16.40.203:2181
    查看 zk 的运行状态
    bin/zkServer.sh status 172.16.40.201:2181
bin/zkServer.sh status 172.16.40.202:2181
bin/zkServer.sh status 172.16.40.203:2181
    节点管理
  7. ls path: 查看当前 ZooKeeper 中所包含的内容
  8. create /zk myData: 创建一个新的 znode。这个命令创建了一个新的 znode 节点“ zk ”以及与它关联的字符串。
  9. get path: 获取该znode信息
  10. set path data: 给znode设置值(前提是值存在)
  11. delete path: 删除该znode

传递四个字母的字符串给ZooKeeper，ZooKeeper会返回一些有用的信息。例如:

echo envi |nc 172.16.40.201 2181
ZooKeeper四字命令 功能描述
conf 输出相关服务配置的详细信息。
cons 列出所有连接到服务器的客户端的完全的连接/会话的详细信息。包括“接受/发送”的包数量、会话id、操作延迟、最后的操作执行等等信息。
dump 列出未经处理的会话和临时节点。
envi 输出关于服务环境的详细信息（区别于conf命令）。
reqs 列出未经处理的请求
ruok 测试服务是否处于正确状态。如果确实如此，那么服务返回“imok”，否则不做任何相应。
stat 输出关于性能和连接的客户端的列表。
wchs 列出服务器watch的详细信息。
wchc 通过session列出服务器watch的详细信息，它的输出是一个与watch相关的会话的列表。
wchp 通过路径列出服务器watch的详细信息。它输出一个与session相关的路径。
